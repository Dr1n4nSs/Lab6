# Сыпачев Иван КМБ-2 Лабараторная №6
# Задание 1
## Задача 10
### Текст задачи
Разработать класс с двумя логическими полями. Создать конструктор копирования. Разработать
метод, вычисляющий конъюнкцию полей. Перегрузить метод ToString() для формирования
строки из полей класса. Реализовать дочерний класс (его содержание предложить
самостоятельно и описать в решении: какой содержательный смысл имеют поля; реализовать
конструкторы; предложить и реализовать 2-3 метода). Протестировать все конструкторы и
другие методы базового и дочернего классов.
### Алгоритм решения
Смысл класса - это представление аттестации в ГИБДД. Первое поле отображает статус сдачи теоретического экзамена, а второе - практического.
Базовый класс ExamStatus:
Поля и свойства: Содержит две логические переменные _firstValue и _secondValue, которые представляют пару состояний. Доступ к ним реализован через публичные свойства.
Конструктор копирования: Реализует механизм создания дубликата объекта, перенося значения полей из существующего экземпляра в новый.
Метод CalculateConjunction: Выполняет логическую операцию «И» (конъюнкцию) над двумя значениями, возвращая true только в случае, если оба состояния истинны.
Дочерний класс DrivingLicenseStatus:
Наследует функционал LogicalPair, адаптируя его под логику получения прав.
Метод PassTheory: Устанавливает первое значение в true, фиксируя успешную сдачу теоретической части.
Метод PassPractice: Реализует проверку зависимости — сдача практики (второе значение) возможна только при условии, что теория уже сдана. В противном случае выводится предупреждение.
Метод IssueLicense: Использует базовый метод конъюнкции для проверки готовности. Если оба экзамена сданы, генерирует случайный номер водительского удостоверения с помощью Random.
Основной блок (Main):
Демонстрирует работу конструктора копирования для архивации текущего статуса и проверяет устойчивость логики программы к попыткам нарушить порядок сдачи (практика до теории).

# Задание 2
## Задача 10
### Текст задачи
Создать класс Tiangle. Определить, существует ли треугольник с
такими длинами сторон. Результат
должен быть типа bool.
Унарные операции:
– вычисляется площадь треугольника, результатом должно быть
вещественное число.
Операции приведения типа:
double (неявная) – результатом является периметр треугольника;
bool (явная) – результатом является true, если треугольник с такими
длинами сторон существует, и falseв противном случае.
Бинарные операции:
< > - сравниваются площади треугольников
### Алгоритм решения
Программа реализует модель треугольника, инкапсулирующую данные о сторонах и предоставляющую механизмы проверки существования, расчета площади и перегрузки операторов.
Класс Triangle:
Поля и свойства: Содержит три закрытых поля a, b, c для хранения длин сторон. Доступ к ним осуществляется через публичные свойства с валидацией: если вводимое значение меньше или равно нулю, выводится сообщение об ошибке.
Конструктор: Инициализирует объект, присваивая значения сторонам через свойства, что гарантирует первичную проверку данных.
Вспомогательный статический метод ReadDouble:
Организует цикл для чтения данных из консоли.
Использует double.TryParse для безопасного преобразования строки в число.
Рекурсивное поведение (через цикл): Если ввод некорректен, выводит сообщение об ошибке и запрашивает данные повторно до тех пор, пока не будет введено число.
Логические методы:
Exists: Проверяет неравенство треугольника (сумма любых двух сторон должна быть больше третьей). Возвращает true, если фигура может существовать.
GetArea: Вычисляет площадь по формуле Герона. Если треугольник не существует, метод возвращает 0.
Перегрузка операторов и приведение типов:
Унарный оператор -: Возвращает площадь треугольника.
Неявное приведение (implicit operator double): Позволяет использовать объект треугольника там, где ожидается число с плавающей точкой (возвращает площадь).
Явное приведение (explicit operator bool): Позволяет проверить существование треугольника путем явного приведения типа.
Бинарные операторы < и >: Реализуют логику сравнения двух треугольников на основе их площадей.
Основной блок (Main):
Последовательно запрашивает параметры для двух треугольников, используя ReadDouble.
Демонстрирует работу всех перегруженных операторов: проверяет существование, вычисляет площади и сравнивает объекты между собой, выводя результаты в консоль.
